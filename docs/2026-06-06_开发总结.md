## 一、项目重命名

- **Ultimate Agent** / **桌面 AI 助手 Ultimate** → **Orca Code**
- 包名保留 `ultimate_agent`（避免破坏所有 import 路径）

## 二、模块拆分

`ultimate_agent.py` (4644行) → `ultimate_agent/` 包 (12 模块)

```
ultimate_agent/
├── config.py            # 配置、全局变量、缓存、客户端
├── utils.py             # 编码、路径、token、清理
├── security.py          # 危险模式、URL 校验、技能沙箱
├── tools_core.py        # 命令执行、文件读写、搜索
├── tools_office.py      # Excel、Word、截图、OCR
├── tools_web.py         # 网页、搜索、天气、定位
├── tools_dev.py         # Git、代码导航、视觉、摄像头
├── tools_skills.py      # 技能系统 + 定时任务
├── tools_automation.py  # GUI 自动化 (7 个工具)
├── tts_mcp.py           # TTS、语音输入、MCP 协议
├── session.py           # 会话、提示词、消息、API
├── main.py              # 工具注册表、用户输入、主循环
```

`ultimate_agent.py` 变为 10 行兼容 shim。

## 三、安全策略变更

### 放宽（第 10-12 批）

| 之前拦截 | 现在放行 |
|----------|----------|
| `del`/`move`/`ren`/`mkdir`/`rmdir`/`set` | ✅ 全部通过 cmd /c |
| `&` `\|` `;` shell 元字符 | ✅ 全部放行 |
| `curl \| bash` / `shutdown` / `Invoke-WebRequest` | ✅ 全部放行 |
| SSRF 防护（内网/localhost URL） | ✅ 仅校验 HTTP/HTTPS 协议 |
| `.py` 文件禁止写入 | ✅ 放行 |
| `tokenizer`/`.venv`/`models` 目录保护 | ✅ 放行 |
| `execute_python` 拦截 `os`/`subprocess`/`sys`/`ctypes` | ✅ 放行（仅保留 eval/exec/compile 拦截） |
| `execute_python` 拦截所有 import | ✅ 白名单模式（仅拦截危险模块） |

### 保留的底线

- `format` — 格式化磁盘
- `rm -rf /` — 递归删除根目录
- `dd if=... of=/dev` — 直接写裸设备

## 四、命令执行重构

### PowerShell 支持
- `powershell` / `pwsh` 加入直接执行通道
- `Start-Process` 替代 `start` 启动 GUI 应用
- `Get-ItemProperty HKLM:\...\App Paths\` 注册表查询替代全盘 `search_files`

### .exe 自动包装
- 直接运行 `"D:\...\app.exe"` → 自动重定向为 `cmd /c start "" "D:\...\app.exe"`
- GUI 启动自动关闭 `CREATE_NO_WINDOW` flag

### 编码修复
- `cmd /c` 输出 → 系统编码 (GBK)，不再乱码
- Python/git 子进程 → UTF-8
- `search_content` findstr → 系统编码

## 五、GUI 自动化重构

### 新增工具

| 工具 | 实现 | 替代的手写代码 |
|------|------|:--:|
| `gui_hotkey` | `pyautogui.hotkey(*keys)` | `execute_python` 手写热键 |
| `gui_press` | `pyautogui.press(key)` | 逐字符模拟 |
| `window_focus` | `pygetwindow.getWindowsWithTitle` + `activate()` | 窗口枚举+激活 |
| `find_on_screen` | `mss` 截图 + `RapidOCR` 坐标识别 | PIL 像素扫描 |

### 输入方式变更

| 版本 | 方法 | UWP 兼容 |
|------|------|:--:|
| v3.0 | `pyautogui.typewrite` 逐字符 | ❌ |
| v4.0 初版 | `pywinauto` UIA `type_keys` | ❌ (假报成功) |
| v4.0 终版 | `pyperclip.copy` + `Ctrl+V` 粘贴 | ✅ |

### 其他变更
- 取消 GUI 操作确认弹窗 (`_gui_confirm` → 直接返回 `True`)
- `gui_type` 拦截热键模式（`{win}`/`+s` 等）并引导使用 `gui_hotkey`

## 六、视觉 API 修复

- `analyze_image` 独立视觉客户端补充 `timeout=300s`
- 请求级 `timeout=60s` 防止卡死
- `analyse_image` 别名注册（英式拼写容错）
- `capture_camera` 输出从系统临时目录改为 `output/`

## 七、截图 & 文件输出

- `take_screenshot` 默认路径 → `output/screenshot_xxx.png`
- 绝对桌面路径自动重定向到 `output/`
- `capture_camera` 输出 `output/camera_xxx.jpg`
- 系统提示词明确：文件输出一律用相对路径到 `output/`

## 八、API & 消息修复

- `vision_client` 补充 `openai.Timeout(300.0, connect=10.0)`
- `sanitize_messages` / `call_model` 双重删除空 `tool_calls: []`（DeepSeek API 拒绝空数组）
- 读取超时从 60s 提高到 300s（适配思考模式）
- Ctrl+C 中断提示加入欢迎界面

## 九、循环导入解决

7 处惰性导入解决模块拆分后的循环依赖：

| 模块 | 导入 | 方式 |
|------|------|------|
| `session.py` | `TOOLS`, `TOOL_MAP`, `run_tool` | `_get_tools()` / `from main import` |
| `tts_mcp.py` | `TOOLS`, `TOOL_MAP` | `init_mcp_tools` 内 `from main import` |
| `tools_skills.py` | `TOOL_MAP`, `session` | 函数内惰性导入 |
| `tools_web.py` | `_TEST_LOCATION_HASH` | 从 security 显式导入 |
| `tools_core.py` | `CONFIG_JSON` | 从 config 显式导入 |
| `main.py` | `_loaded_skills`, `_autoload_skills_cache` 等 | 从 skills 显式导入 `_` 前缀变量 |

## 十、系统提示词演进

| 规则 | 目的 |
|------|------|
| 启动软件 → PowerShell `Start-Process` | 替代 Win+R 模拟按键 |
| 找不到 exe → `Get-ItemProperty` 注册表 | 替代全盘 `search_files` |
| 文件输出 → `output/` 相对路径 | 替代桌面污染 |
| 看屏幕布局 → `analyze_image` | 视觉模型 |
| 读终端文本 → `ocr_image` | 本地 OCR |
| 操作桌面应用 → `window_focus` → `find_on_screen` → `gui_click` | 替代像素扫描 |
| 发消息不验证 → 信任 Enter | 替代 3 像素误差误判 |
