# Orca Code v5.2 开发总结

> 日期：2026-06-09 | 版本：v5.2.0 | TUI 生产就绪

## TUI 架构 — FullscreenLayout 模式

借鉴 Claude Code 的 `FullscreenLayout.tsx`，完成 `app.tsx` 架构级重写：

```
Root (height=rows)
├── Banner (flexShrink=0)        ← 永不压缩
├── Welcome/Scroll (flexGrow=1)  ← 吸收剩余空间
│   └── overflow="hidden"        ← 溢出裁剪
└── BottomBar (flexShrink=0)     ← 永不挤压
    ├── Status (spinner/tokens/error)
    ├── Divider
    └── Input ("你 › ")
```

### 关键设计决策

1. **`flexShrink={0}`** — banner、welcome、bottom-bar 三个区域都不会被压缩
2. **`flexGrow={1}`** — 滚动区吸收所有剩余空间
3. **`overflow="hidden"`** — 历史消息溢出时裁剪而不是撑爆布局
4. **Welcome 占位** — `isWelcome` 时加 `<Box flexGrow={1} />` 把输入栏推到底部

## 消息顺序修正

**问题**：模型推理 → 工具调用 → 最终回答，但 TUI 显示为「推理+回答」在上、「工具」在下。

**根因**：`flushFinalToMessages` 把思考+回复塞进一条消息，插入在 tools 前面。

**修复**：拆分为两条消息：
- 思考 (reasoning) → 插在 user 后、tools 前
- 回答 (content) → 追加在 tools 后

**最终显示**：`💭 思考…` → `⚡ 工具` → `● 回复`

## 流式与工具进度

1. **`tool_executing` 增强** — 后端新增 `tools: string[]` 字段，前端显示 `⚡ search_web, read_webpage …` 而非死板计数
2. **`streamTextRef` 清零** — `tool_executing` 时清空，每个 agent-loop turn 的文本独立，不再跨 turn 拼接
3. **`streamReasoningRef` 不清** — 推理跨 turn 累积是合理的（模型持续思考）

## 输入修复

- **退格 (Windows)** — 物理退格发送 `0x7F` (DEL)，Ink 的 `parseKeypress` 映射为 `key.delete=true`。Handler 同时检查 `key.backspace`、`key.delete`、原始字节 `\x7f`、`\b`、`\x08`
- **IME 光标** — `\x1b[?25h` (DECTCEM) 在组件挂载后显示终端光标，IME 组合窗口正确定位
- **稳定 handler** — `useCallback([], [])` + `useRef` 防止 Ink 的 listener churn

## Banner 优化

- 从 468 个 `<Text>`（每字符一个）简化到 6 个（每行一个）
- 预计算 `BANNER_COLORS` 数组，memo 渲染
- `v5.2` 版本号不再合并到 banner 末尾行

## Token 与 UI 细节

- Token 0/0 时隐藏（API 不返回 usage info）
- `⏱` emoji 替换为 `[Ns]`（避免终端双宽字符重叠）
- reasoning-only 消息不显示空 `● …`

## 后端

- **Agent loop**: 模型调工具 → 执行 → 结果反馈 → 循环（max 10 turns）
- **YOLO 强制**: server 模式下 `PermissionMode.YOLO`，避免 `msvcrt.getwch()` 阻塞
- **健康检查**: 从 `/health` 修正为 `/v1/health`
- **一键启动**: `start_all.bat` 带 PowerShell 健康检查轮询
