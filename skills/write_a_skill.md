## 协议：创建技能

### 流程

1. **收集需求** — 问用户：
   - 技能覆盖什么任务/领域？
   - 哪些具体用例？
   - 需要可执行脚本还是纯指令？
   - 需要哪些参考材料？

2. **起草技能** — 创建 `.md` 文件：
   - 符合当前技能加载格式（YAML frontmatter + markdown body）
   - 包含 `triggers` 字段（至少 3-5 个中文触发词）
   - 内容控制在 200 行以内，太长考虑拆分引用文件

3. **给用户审核** — 问：
   - 覆盖了用例吗？
   - 有没有漏掉或不清的地方？
   - 需要增加/减少细节？

### 技能结构

```markdown
---
name: skill-name
description: 一句话说明技能能力。后面加"触发方式：关键词列表"。
triggers:
  - /触发词
  - 自然语言触发
  - 更多触发方式
---

## 协议：技能名称

清晰的步骤、原则、检查清单。
```

### 描述编写要点

描述是 Agent 选择技能的**唯一依据**。系统提示词里所有技能描述排在一起，Agent 根据用户输入匹配合适技能。

- 第一句：做什么
- 第二句：什么时候用（列出典型触发场景）
- 不超过 200 字

### 审核清单
- [ ] description 包含触发条件说明
- [ ] triggers 有 3+ 个接地气的中文短语
- [ ] 整体不超过 200 行
- [ ] 不含时效性信息
- [ ] 术语一致
- [ ] 包含具体例子
